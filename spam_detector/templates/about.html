About Spam Mail Detector

An academic machine learning project Semester Spring 2026.

Project Overview

Spam Mail Detector is an AI-powered system developed as a semester project for Artificial Intelligence. It demonstrates a complete machine learning pipeline, including data preprocessing, feature extraction, model training, evaluation, and deployment. The project is integrated with a simple and interactive web application that allows users to easily check whether an email is spam or not in real time.

Technical Stack

🐍Python 3
🔬Scikit-learn
🌐Flask
🗂️Pandas / NumPy
📊Matplotlib / Seaborn
💾Joblib (Model Serialisation)

AI Concepts Applied

  • Text Preprocessing — Lowercasing, URL normalisation, punctuation and digit removal.
  • Feature Extraction — TF-IDF vectoriser with uni-gram and bi-gram features (max 5,000 features), sublinear TF scaling.
  • Supervised Classification — Logistic Regression with L2 regularisation (C=1.0).
  • Model Evaluation — Accuracy, Precision, Recall, F1-Score, and Confusion Matrix on a held-out 20% test set.
  • Deployment — REST API via Flask served to a responsive HTML/CSS/JS front end.

Pipeline

  1. Raw email CSV loaded → subject + body combined into single text field.
  2. Text cleaned: lowercase, URL tokens, digit tokens, punctuation stripped.
  3. 80/20 stratified train/test split to preserve class balance.
  4. TF-IDF vectoriser fit on training data only — applied to test data.
  5. Logistic Regression trained; model artefacts persisted with Joblib.
  6. Evaluation charts generated and saved as static images.
  7. Flask app loads model at start-up; serves predictions via /api/predict.

Challenges & Solutions

Challenge

Avoiding data leakage between training and inference preprocessing.

Solution

The TF-IDF vectoriser is fit only on training data; the same fitted object is serialised and reused at inference time.

Challenge

Providing meaningful confidence instead of a binary label.

Solution

Logistic Regression's predict_proba is exposed directly so users see exact spam / ham probabilities.

Challenge

Making evaluation results visible to non-technical users.

Solution

A dedicated dashboard page renders all metrics, charts, and a labelled confusion matrix in plain English.
